# Обзор платформы для разработчиков

TrucklineMP предлагает платформу для разработчиков для сторонних интеграций. Вы можете читать публичные данные платформы через **Public API**, идентифицировать пользователей с помощью **OAuth 2.0** и получать обновления в реальном времени через **вебхуки**.

Эта документация охватывает концепции и рабочие процессы. Для получения полного списка эндпоинтов, форматов запросов и схем ответов используйте интерактивный справочник по API на главном сайте.

## Что вы можете создать

| Интеграция | Лучше всего подходит для |
|-------------|----------|
| **Public API** | Ботов, панелей управления, сайтов со статистикой и инструментов, которые читают данные VTC, мероприятий, новостей, пользователей и модерации |
| **OAuth apps** | Приложений, которые авторизуют пользователей через их аккаунт TrucklineMP и читают данные профиля в рамках выделенных областей доступа |
| **Webhooks** | Сервисов, которые реагируют на события платформы (вступления участников, баны, ответы на приглашения и многое другое) |

## Перед тем как начать

1. Создайте аккаунт TrucklineMP на сайте [trucklinemp.com](https://trucklinemp.com).
2. Включите **Developer Mode** по пути **Account Settings → Security → Developer Mode → Activate**.
3. Откройте [Консоль разработчика](https://trucklinemp.com/developer) из панели навигации или настроек безопасности.

Режим разработчика открывает доступ к консоли. Сам по себе он не предоставляет расширенных разрешений API. Публичные эндпоинты API остаются доступными только для чтения. Области доступа OAuth контролируют, к каким пользовательским данным ваше приложение может получить доступ после того, как пользователь его авторизует.

## Быстрый старт: Публичный API

1. В Консоли разработчика создайте **проект** (project).
2. Сгенерируйте ключ API (префикс `tl_...`). Скопируйте его, когда он отобразится. Позже его нельзя будет просмотреть.
3. Отправьте запрос с вашим ключом:

```bash
curl -H "Authorization: Bearer tl_YOUR_API_KEY" \
  "https://api.trucklinemp.com/vtcs"
```

4. Опробуйте эндпоинты в интерактивном режиме в [API Playground](https://trucklinemp.com/developer/playground).

Публичные эндпоинты для чтения работают **без** ключа API. Ключи повышают ваши лимиты запросов и привязывают использование к вашему проекту для аналитики.

## Быстрый старт: Приложение OAuth

1. В Консоли разработчика перейдите в раздел **OAuth Apps** и создайте приложение.
2. Укажите **redirect URIs** и **scopes** на странице настроек OAuth.
3. Скопируйте ссылку для установки **install link** со страницы обзора приложения (или создайте собственный URL авторизации).
4. Пока приложение не опубликовано, добавьте тестовых пользователей **test users**, чтобы другие могли авторизовать его во время разработки.
5. Когда всё будет готово, пройдите верификацию домена и опубликуйте приложение через страницу публикации.

Смотрите раздел [Приложения OAuth](/ru/guides/developers/oauth-apps/) для получения полной информации о потоке авторизации, PKCE и правилах публикации.

## Конфиденциальность и соответствие требованиям

Использование API логируется для ограничения количества запросов, предотвращения злоупотреблений и аналитики. О том, как обрабатывается телеметрия, читайте в [Политике конфиденциальности](https://id.trucklinemp.com/legal/privacy).

Приложения OAuth обязаны предоставлять ссылку на **политику конфиденциальности** и должны предоставлять ссылку на **условия использования** перед публикацией.
